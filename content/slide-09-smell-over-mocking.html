Smell 2 Over-Mocking

Bad example

@Test
void processOrder_ShouldSucceed() {
    User mockUser = mock(User.class);
    when(mockUser.isActive()).thenReturn(true);

    orderService.process(mockUser, order);
}

Good example

@Test
void processOrder_ShouldSucceed_ForActiveUser() {
    // Arrange
    User user = new User(1L, "John", Status.ACTIVE);

    // Act
    OrderResult result = orderService.process(user, order);

    // Assert
    assertEquals(SUCCESS, result.status());
}

Mock only boundaries

Use mocks for expensive, slow, external, or non-deterministic collaborators.

Do not mock domain truth

If the object is simple and in-memory, prefer a real instance.
