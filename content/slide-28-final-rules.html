If You Remember Only This

  1. Test behavior, not implementation.
  2. Assert state, not interactions.
  3. Avoid logic in tests.
  4. Prefer DAMP over DRY.
  5. Use meaningful assertions.
  6. Keep tests deterministic.
  7. Test through public APIs.
  8. Prefer readability over cleverness.
  9. Tests should survive harmless refactors.
