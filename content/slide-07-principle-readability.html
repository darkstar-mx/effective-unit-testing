Principle 3 Readable Tests Are Reviewable Tests

Bad example

@Test
void test1() {
    helper(true, false, "Standard");
}

Good example

@Test
void shouldProcessOrder_WhenUserIsActive() {
    User user = new User("John", Status.ACTIVE);
    Order order = new Order(100);

    Result result = service.process(user, order);

    assertEquals(Result.SUCCESS, result.status());
}

Write for review

The name, setup, and assertion should tell the story without extra explanation.

Prefer DAMP

Descriptive duplication is often cheaper than hiding intent behind helpers.
