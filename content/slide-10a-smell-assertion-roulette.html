Smell 4 Assertion Roulette

Bad example

@Test
void createInvoice_ShouldPopulateFields() {
    Invoice invoice = service.createInvoice(order);

    assertEquals("INV-001", invoice.id());
    assertEquals(CREATED, invoice.status());
    assertEquals(customerId, invoice.customerId());
    assertEquals(3, invoice.lines().size());
    assertEquals(total, invoice.total());
}

Good example

@Test
void createInvoice_ShouldReturnCreatedInvoiceForOrder() {
    Invoice invoice = service.createInvoice(order);

    assertAll("invoice",
        () -> assertEquals("INV-001", invoice.id(), "invoice id"),
        () -> assertEquals(CREATED, invoice.status(), "invoice status"),
        () -> assertEquals(customerId, invoice.customerId(), "customer"),
        () -> assertEquals(3, invoice.lines().size(), "line count"),
        () -> assertEquals(total, invoice.total(), "invoice total")
    );
}

Problem

When one assertion fails, the test output often hides the rest of the intent.

Fix

Group related checks, add messages, or compare richer expected objects when the domain allows it.
