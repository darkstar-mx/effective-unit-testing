The “Mary” Scenario Why Your CI Is a Liability

1

Senior engineer implements a 20-line feature.

2

CI returns 40 unrelated failures.

3

Failures are brittle tests reacting to harmless internal changes.

@Test
void updateInternalState_ShouldCallPrivateMethod() {
    // Testing implementation details
}
@Test
void deposit_ShouldIncreaseBalance_WhenAmountIsPositive() {
    Account account = new Account(100);

    account.deposit(50);

    assertEquals(new Account(150), account);
}

Tests should fail only when behavior changes.
