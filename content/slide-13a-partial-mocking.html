Partial Mocking Useful, but a Warning Sign

Bad example

@Test
void generateReport_ShouldSucceed() {
    ReportService service = partialMock(new ReportService());

    when(service.loadExchangeRates()).thenReturn(fixedRates);

    service.generateReport(order);
}

Good example

@Test
void generateReport_ShouldUseProvidedExchangeRates() {
    // Arrange
    ExchangeRateProvider rates = () -> fixedRates;
    ReportService service = new ReportService(rates);

    // Act
    Report report = service.generateReport(order);

    // Assert
    assertEquals(EXPECTED_TOTAL, report.total());
}

Why it is risky

Partial mocking usually means the object has mixed responsibilities and no clean seam.

When acceptable

Use it sparingly for legacy code under change, and treat it as a refactoring target rather than an end state.

Prefer extracting collaborators over overriding part of the object under test.
