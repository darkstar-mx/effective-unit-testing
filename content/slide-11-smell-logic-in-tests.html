Smell 4 Logic Hidden Inside Tests

@Test
void calculateTax_ShouldBeCorrectForAllRegions() {
    List<String> regions = List.of("NY", "CA", "TX");

    for (String region : regions) {
        assertTrue(service.calculate(100, region) > 0);
    }
}
@ParameterizedTest
@ValueSource(strings = {"NY", "CA", "TX"})
void calculateTax_ShouldReturnPositiveValue(String region) {
    assertTrue(service.calculate(100, region) > 0);
}

Tests should be easy to read linearly. If a test needs its own logic, the intent is being buried.
