Mocking Rules Use Interaction Testing Deliberately

Good candidates

  • HTTP clients
  • Message brokers
  • Databases and gateways
  • Clock, UUID, randomness

Bad candidates

  • Value objects
  • Entities with simple state
  • Collections
  • Your own domain rules
@Test
void submit_ShouldSendExpectedPayloadToGateway() {
    service.submit(order);

    verify(gateway).send(expectedRequest(order));
}

Interaction assertions are correct when the interaction itself is the contract.
