Technique AAA Makes Intent Obvious

Bad example

@Test
void testProcess() {
    Service service = new Service();
    service.init();
    assertTrue(service.isValid());

    Response result = service.process(new Request(10));
    assertEquals(20, result.val());
}

Good example

@Test
void process_ShouldDoubleValue_WhenInputIsPositive() {
    // Arrange
    Service service = new Service();
    Request request = new Request(10);

    // Act
    Response actual = service.process(request);

    // Assert
    assertEquals(new Response(20), actual);
}

Why it matters

AAA reduces hidden setup, mixed concerns, and review friction.

Review rule

If Arrange, Act, and Assert are blurred together, ask for a rewrite.
