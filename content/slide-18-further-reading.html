Production code smells

  • Long Method, Large Class, Feature Envy, Data Clumps, Shotgun Surgery, Duplicate Code
  • Primary reference: Refactoring.Guru
  • Catalog: https://refactoring.guru/refactoring/smells
  • Example detail page: https://refactoring.guru/smells/feature-envy

How to use these references

  • Use the smell name in PR comments.
  • Describe the impact on trust, readability, or maintainability.
  • Suggest a concrete rewrite or refactoring, not just the label.

Team habit to build

A shared vocabulary for smells makes review comments faster, less subjective, and easier to act on.
