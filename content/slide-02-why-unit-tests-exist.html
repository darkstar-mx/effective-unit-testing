Why Unit Tests Exist And Why They Fail

Problem

  • Teams believe tests are about “coverage” or “correctness”.
  • Tests become a burden instead of a tool.
  • Engineers start avoiding or deleting tests.

Rule of Thumb

If your code is hard to test, your design is the problem.

class OrderService {
    double calculateTotal(Order order) {
        double base = order.getItems()
            .stream()
            .mapToDouble(Item::price)
            .sum();

        return base * 1.16;
    }
}
class OrderService {
    private final TaxPolicy taxPolicy;

    OrderService(TaxPolicy taxPolicy) {
        this.taxPolicy = taxPolicy;
    }

    double calculateTotal(Order order) {
        return taxPolicy.apply(order.subtotal());
    }
}

Good tests improve design, make refactoring safe, and reduce long-term maintenance cost.
