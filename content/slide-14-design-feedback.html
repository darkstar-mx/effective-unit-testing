Hard to Test Often Means Hard to Design

class OrderService {
    double calculateTotal(Order order) {
        double base = order.getItems()
            .stream()
            .mapToDouble(Item::price)
            .sum();

        return base * 1.16;
    }
}
class OrderService {
    private final TaxPolicy taxPolicy;

    OrderService(TaxPolicy taxPolicy) {
        this.taxPolicy = taxPolicy;
    }

    double calculateTotal(Order order) {
        return taxPolicy.apply(order.subtotal());
    }
}

Good unit testing is not only verification. It is a design feedback loop that pushes dependencies, policy, and side effects to clear seams.
